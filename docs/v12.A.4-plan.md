# v12.A.4 实施 plan — Market Regime + 结构化复盘

> 落地日期: 2026-06-16
> 借鉴来源: felix-quant 借鉴点 #1 (5 档市场状态 + 决策模式) + #3 (结构化复盘)
> 详细分析: `docs/借鉴分析-felix-quant.md`

## 目标
1. **借鉴 #1**: 引入 5 档市场状态 (Panic/RiskOff/Choppy/RiskOn/Recovery) + 4 档决策模式 (WAIT/DEFENSIVE_OBSERVE/WATCH/PROBE), 替代当前"v3 拍脑袋选 A/B/C"
2. **借鉴 #3**: 引入 `reviews` 表 + `add_review` / `query_reviews` skill + temporal_facts `REVIEWED` predicate

## 改动文件清单

### 借鉴 #1 — Market Regime + Decision Engine
- **新文件** `stock_trading_agent/engine/market_regime.py` (~200 行)
  - 5 档分类 `classify_regime(env: dict) -> str`
  - 4 档决策 `decide(regime, candidate_count=0, ...) -> {mode, position_min, position_max, allowed, forbidden, reasons}`
- **新文件** `stock_trading_agent/engine/decision_engine.py` (~150 行)
  - `build_decision(regime, signals=[], market_context={}) -> dict`
  - 仿 felix: regime → mode 映射 + 4 维仓位上限 (base/market/strategy_quality/decision_mode)
- **改** `stock_trading_agent/engine/skills.py`
  - `_run_get_market_env`: result 加 `regime` 字段 (5 档), `decision_mode`, `position_min/max`
  - 新 skill `get_daily_decision` — 飞书问"今天能买吗/今日决策"
- **改** `stock_trading_agent/agent/stages.py`
  - `stage_pick`: 跑选股前先 `build_decision()`, Panic 模式直接 return 不选; 其他模式把 `position_max` 当 max_position_ratio 传给 picker
- **改** `stock_trading_agent/feishu/card_templates.py` (or engine/cards.py)
  - 加 `card_decision(payload)` 渲染 regime + mode + 仓位 + 理由
- **测试** `tests/test_v12_a_4_regime.py` (10 个)

### 借鉴 #3 — Reviews 表 + Skill
- **改** `stock_trading_agent/engine/paper_trader.py`
  - SCHEMA 加 `reviews` 表 (仿 felix): date/stock_code/signal_id/action_taken/reason/result/summary/tags
  - 老库 ALTER 兼容 (CREATE IF NOT EXISTS)
- **新文件** `stock_trading_agent/engine/reviews.py` (~120 行)
  - `add_review(date, stock_code, *, signal_id=None, action_taken=False, reason, result, summary, tags=[]) -> int`
  - `query_reviews(date=None, stock_code=None, tag=None, limit=20) -> list[dict]`
  - `tag_count(days=30) -> dict[tag, count]` (统计高频复盘标签)
- **改** `stock_trading_agent/engine/temporal_facts.py`
  - PREDICATE_VOCAB 加 `REVIEWED` (跟 SELECTED/VALIDATED 平级)
- **新 skill** `add_review` + `query_reviews` (在 `engine/skills.py` 注册)
  - `add_review` 接受自然语言: `"加复盘: 002063 止盈 2.5% 早盘冲高 题材退潮"` → 解析 → 落 reviews 表 + 写 REVIEWED fact
  - `query_reviews` 按 date/code/tag 查
- **改** `stock_trading_agent/feishu/admin_cmd.py` (or 新 CLI 子命令)
  - `agent review add/list` 类似 `agent memory list/clear`
- **测试** `tests/test_v12_a_4_reviews.py` (8 个)

## 总计
- 新文件 4 个 (market_regime.py / decision_engine.py / reviews.py + 1 skill 集成到 skills.py)
- 改文件 5 个 (skills.py / stages.py / paper_trader.py / temporal_facts.py / card_templates.py)
- 新测试 2 套件 ~18 个
- 预计代码量: +600 行, 改 ~150 行

1. 跑 `pytest tests/test_v12_a_4_*.py` 全绿
