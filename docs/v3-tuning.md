## 2. 调参决策（v3 落地值）

### 2.1 评分上限 `SCORE_MAX = 80.0`

| 项 | 详情 |
|---|---|
| 旧值 | 无上限 |
| 新值 | **80.0** |
| safe_range | [75.0, 85.0] |
| 依据 | ≥80 段胜率 17% 远低于 75-80 段的 62% (n=23 vs 8)，历史上是反向指标 |
| 预期效果 | 自动剔除"超涨高位票"，预计每日 5-15% 的高分票被过滤 |
| 撤回条件 | 某周 ≥80 段胜率 ≥60% 且 n≥5 |

### 2.2 强信号带加分 `STRONG_BAND = 3.0-3.5% + 5 分`

| 项 | 详情 |
|---|---|
| 旧值 | 无强信号带（涨幅越接近 3% 评分越高，但没额外加权） |
| 新值 | **3.0-3.5% + 5 分**（落在带内 +5，不在带内 0） |
| safe_range（lo） | [2.8, 3.2] |
| safe_range（hi） | [3.3, 3.7] |
| 依据 | 3.0-3.5% 段胜率 44% >> 3.5-4.0% 段 25% (n=16 vs 16) |
| 预期效果 | 强信号带内的票排名靠前被选，3.5-4.0% 的边缘票被压后 |
| 边界观察 | 6/5 复盘时 4.5-4.8% 区间有 2 只危险边缘票（京东方/蓝思） |

### 2.3 板块黑名单 7 个板块

| 板块 | 样本数 | 平均次日中午 | 胜率 | 决策 |
|---|---:|---:|---:|---|
| 光伏设备 | 2 | -3.23% | 0% | **加入** |
| 电子化学品Ⅱ | 2 | -1.46% | 0% | **加入** |
| 通用设备 | 4 | -1.44% | 0% | **加入** |
| 塑料 | 2 | -1.23% | 50% | **加入**（avg 差） |
| 消费电子 | 3 | -1.30% | 33% | **加入** |
| 电池 | 4 | -0.75% | 50% | **加入**（avg 差） |
| 软件开发 | 3 | -0.55% | 33% | **加入** |

- 每次最多 +2 / -2 板块（防单周剧烈变化）
- `safe_sectors: []`（v1 没有永久保护板块）
- 2 周后跑一次 `_compute_review_stats` 看这些板块是否仍拖累，拖累反转就移出

## 3. 调参执行机制

### 3.1 流程
```
周日 20:00 cron
  → reviewer.run_weekly_review()
  → tuner.run_weekly_tune()
    → weekly_stats() 拉本周数据
    → _propose_score_max / _propose_strong_band / _propose_blacklist
    → apply_proposal(auto=True)  // safe_range 内
    → pending[]  // safe_range 外，推飞书确认
  → 推飞书周报卡片（含 applied + pending）
```

### 3.2 安全范围策略

| 维度 | 行为 |
|---|---|
| 数值在 safe_range 内 | 周日自动改 `config.yaml` + 写 `params_history` |
| 数值在 safe_range 外 | 推飞书卡片「提议改动 X → Y，依据 Z」+ 按钮 [应用] [拒绝] |
| 黑名单 +/- 超 2 | 走飞书确认 |
| 用户 7 天未确认 pending | v1 暂不自动应用，v2 改为 "48 小时未响应 = 拒绝" |

### 3.3 params_history 表

```sql
SELECT * FROM params_history ORDER BY changed_at DESC LIMIT 20;
```

每次自动改 + 人工确认都会留痕，方便日后审计"那次胜率崩盘是不是某次调参"。
